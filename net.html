Network Security Simulation Guide

Welcome to the Network Security Simulation! This simulation will help you understand how to configure and manage network services on a Linux machine. Follow the steps below to get started:

  1. Type commands in the terminal input field and press Enter to execute them.
  2. Type help in the commands terminal input and press enter to view all commands.
  3. Use commands like nmap, netstat, service, apache2ctl, ssh-keygen, and /etc/init.d/ssh start to interact with the system.
  4. Observe the output in the terminal to understand the status of various services and network configurations.
  5. Please note that not all available commands are explained in the step-by-step instructions. Please type "help" in the command inteface to see additional commands.

Step-by-Step Instructions

  1. **Scan for open ports**:
    • Type nmap 127.0.0.1 and press Enter.
    • Observe the output to see which ports are open.
  2. **View running services**:
    • Type netstat -tan and press Enter.
    • Observe the output to see the active internet connections.
  3. **Start VSFTPD service**:
    • Type service vsftpd start and press Enter.
    • Check the output to confirm the service has started.
  4. **Start Apache web server**:
    • Type apache2ctl start and press Enter.
    • Check the output to confirm the web server has started.
  5. **Generate SSH keys**:
    • Type ssh-keygen and press Enter.
    • Hit Enter three times to accept the default settings.
    • Observe the output to confirm the keys have been generated.
  6. **Start SSH service**:
    • Type /etc/init.d/ssh start and press Enter.
    • Check the output to confirm the SSH service has started.
  7. **View running services again**:
    • Type netstat -tan and press Enter.
    • Observe the output to see the updated list of active internet connections.
  8. **Install and start MySQL service**:
    • Type apt-get install mysql-server and press Enter.
    • Follow the prompts to complete the installation.
    • Type service mysql start and press Enter.
    • Check the output to confirm the MySQL service has started.
  9. **Create a new user in MySQL**:
    • Type mysql -u root -p and press Enter.
    • Enter the root password when prompted.
    • Type CREATE USER 'newuser'@'localhost' IDENTIFIED BY 'password'; and press Enter.
    • Type GRANT ALL PRIVILEGES ON *.* TO 'newuser'@'localhost'; and press Enter.
    • Type FLUSH PRIVILEGES; and press Enter.
    • Type EXIT; and press Enter.
  10. **Install and configure UFW (Uncomplicated Firewall)**:
    • Type apt-get install ufw and press Enter.
    • Type ufw allow 22 and press Enter to allow SSH connections.
    • Type ufw allow 80 and press Enter to allow HTTP connections.
    • Type ufw allow 3306 and press Enter to allow MySQL connections.
    • Type ufw enable and press Enter to enable the firewall.
    • Check the output to confirm the firewall rules have been applied.

What You Will Learn

  • How to scan for open ports using nmap.
  • How to view running services using netstat.
  • How to start and manage services like VSFTPD, Apache, SSH, and MySQL.
  • How to generate SSH keys for secure communication.
  • How to create and manage users in MySQL.
  • How to configure and manage a firewall using UFW.

Applying This Knowledge in the Real World

By using this simulation, you will gain practical experience in managing network services on a Linux machine. This knowledge can be applied in various real-world scenarios, such as:

  • Setting up secure file transfer services in a corporate environment.
  • Configuring web servers to host websites and applications.
  • Implementing secure remote access using SSH.
  • Performing network scans to identify potential security vulnerabilities.
  • Managing databases and users in MySQL.
  • Configuring and managing firewalls to enhance network security.

Command Explanations

         
    
             
  • nmap 127.0.0.1: Used to scan for open ports on a remote machine.
    • 
             
  • netstat -tan: Displays active internet connections and listening ports.
    • 
             
  • service vsftpd start: Starts the VSFTPD (Very Secure FTP Daemon) service.
    • 
             
  • apache2ctl start: Starts the Apache web server.
    • 
             
  • ssh-keygen: Generates SSH keys for secure communication.
    • 
             
  • /etc/init.d/ssh start: Starts the SSH service.
    • 
             
  • service mysql start: Starts the MySQL database server.
    • 
             
  • mysql -u root -p: Logs into MySQL as the root user.
    • 
             
  • CREATE USER 'newuser'@'localhost' IDENTIFIED BY 'password'; Creates a new user in MySQL.
    • 
             
  • GRANT ALL PRIVILEGES ON *.* TO 'newuser'@'localhost'; Grants all privileges to the new MySQL user.
    • 
             
  • FLUSH PRIVILEGES; Applies changes made to user privileges in MySQL.
    • 
             
  • apt-get install ufw: Installs the Uncomplicated Firewall (UFW) package.
    • 
             
  • ufw allow 22: Allows SSH connections through the firewall.
    • 
             
  • ufw allow 80: Allows HTTP connections through the firewall.
    • 
             
  • ufw allow 3306: Allows MySQL connections through the firewall.
    • 
             
  • ufw enable: Enables the UFW firewall.
    • 
             
  • df -h: Displays disk space usage.
    • 
            
  • systemctl status apache2: Shows the status of the Apache service.
    • 
            
  • systemctl restart apache2: Restarts the Apache service.
    • 
            
  • top: Shows the top processes running on the system.
    • 
            
  • ps aux: Lists all running processes with their details.
    • 
            
  • useradd newuser: Adds a new user to the system.
    • 
            
  • passwd newuser: Changes the password for the new user.
    • 
            
  • shutdown now: Shuts down the system.
    • 
            
  • reboot: Reboots the system.
