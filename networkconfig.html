Available Commands:

  • netsh interface ipv4 set address name="Ethernet" static 192.168.1.200 255.255.255.0 192.168.1.1 - Configure a static IPv4 address, subnet mask, and default gateway for the "Ethernet" interface.
  • netsh interface ipv4 set dns name="Ethernet" static 8.8.4.4 - Set the primary DNS server to 8.8.4.4 for the "Ethernet" interface.
  • route add 0.0.0.0 mask 0.0.0.0 192.168.1.1 - Add a default gateway route, directing all traffic to 192.168.1.1.
  • route delete 192.168.2.0 - Remove the specific route to the 192.168.2.0 network from the routing table.
  • netstat -an - Display all active network connections and listening ports with their state.
  • netsh int ip reset - Reset TCP/IP settings to their default state, useful for troubleshooting network issues.
  • ipconfig - Display basic network adapter configuration, including IP address and default gateway.
  • ipconfig /release - Release the current DHCP-assigned IP address from the network adapter.
  • ipconfig /renew - Request a new IP address from the DHCP server for the network adapter.
  • ipconfig /all - Display detailed network configuration, including MAC address, DHCP, and DNS settings.
  • ping [IP address] - Send ICMP echo requests to check connectivity and response time from a remote host.
  • tracert [hostname] - Trace the path packets take to reach a destination host, showing intermediate hops.

1. Set Static IP for a Specific Interface

netsh interface ipv4 set address name="Ethernet" static 192.168.1.200 255.255.255.0 192.168.1.1

This command sets a static IP address (192.168.1.200) on the "Ethernet" interface with a subnet mask (255.255.255.0) and a default gateway (192.168.1.1).

2. Set Primary DNS Server

netsh interface ipv4 set dns name="Ethernet" static 8.8.4.4

This command sets the primary DNS server to 8.8.4.4 for the "Ethernet" interface.

3. Add Secondary DNS Server

netsh interface ipv4 add dns name="Ethernet" 8.8.8.8 index=2

This command adds a secondary DNS server (8.8.8.8) to the "Ethernet" interface.

4. Add a Default Route

route add 0.0.0.0 mask 0.0.0.0 192.168.1.1

This command adds a default route to the routing table, using the gateway 192.168.1.1.

5. Delete a Specific Route

route delete 192.168.2.0

This command removes the route to the 192.168.2.0 network from the routing table.

6. Display Active Network Connections

netstat -an

This command lists all active network connections and listening ports.

7. Show Current Network Configuration

ipconfig

This command displays the current network configuration, including the IP address, subnet mask, and default gateway.

8. Show ARP Table

arp -a

This command displays the ARP table, which maps IP addresses to MAC addresses for local network devices.

9. Test Network Connectivity

ping 8.8.8.8

This command pings Google's public DNS server (8.8.8.8) to check network connectivity and response time.
